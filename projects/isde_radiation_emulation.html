Radiation-Effects Emulation Platform

Context: Institute of Space and Defense Electronics (ISDE) — Summer 2024

What it is

A custom ESP32-based hardware emulation platform designed to reproduce radiation-induced behavior in a target electronic component. The system generates controlled analog stimulus and monitors the device response, enabling repeatable lab testing without requiring live irradiation.

Problem

  • Radiation can cause transient or persistent changes in circuit behavior, which are hard to reproduce consistently.
  • Beam time is limited/expensive; we needed a way to test behaviors in the lab without relying on live irradiation for every iteration.
  • The emulation needed to be controlled, measurable, and safe for the device under test.

My contributions

  • Designed a custom PCB integrating an ESP32, DAC, ADC, and voltage level-shifting circuitry to safely interface with the device under test.
  • Developed ESP32 firmware to generate programmable analog stimulus profiles and coordinate synchronized measurements.
  • Implemented ADC-based monitoring to capture device response and characterize behavior under emulated fault conditions.
  • Built a Python-based control and logging workflow to automate test runs and collect repeatable data.

Technical details

  • Controller: ESP32 managing timing, control logic, and communication.
  • Stimulus generation: DAC used to produce controlled analog voltage profiles representing radiation-induced effects.
  • Measurement: ADC used to monitor the response of the device under test in real time.
  • Interface: Voltage level-shifting circuitry to match signal ranges and protect both the ESP32 and the target component.
  • Automation: Python scripts for parameter sweeps, data logging, and repeatable test execution.

Results

  • Enabled repeatable emulation of radiation-induced behavior under controlled conditions.
  • Reduced dependency on live beam testing by supporting rapid bench-level iteration.
  • Improved test consistency through automated stimulus generation and data capture.

What I’d improve next

  • Add calibration routines to compensate for DAC/ADC non-idealities and temperature drift.
  • Increase timing resolution for finer-grained transient emulation.
  • Expand the emulator to support multiple fault profiles and parameterized waveforms.
